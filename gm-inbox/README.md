# gm-inbox · 执行侧呈报信箱（CGM→GM·候审）

**委托人立规（2026-07-21·「给gm审核内容一律统一走信箱」）**：凡执行侧（CGM）呈 GM 审核/签收/评审之内容，**一律以文件落本目录，不走对话粘贴**。与 `adj-inbox/`（GM→CGM 去程）构成双向信箱对。

## 规约
1. **文件名**：`CGM-MMDD-NN.md`（如 CGM-0721-01）·标题行注明呈报性质（签收请求/N15 评审请求/读数申报/议题预呈）。
2. **必备头**：E5 权威四字段 + 呈报日期 + 候裁清单（编号列出请 GM 裁/签的项）。
3. **锚定**：所涉材料一律给 repo 路径 + commit/sha1，GM 直读主仓核验，呈报件本身不复制大段原文（信箱是索引不是货舱）。
4. **回程**：GM 审毕以 ADJ 投递 `adj-inbox/`（照旧三要素）或在呈报件对应回执中签收；CGM 收到回件后将呈报件移入 `gm-archive/`（随回执归档）。
5. **纪律**：呈报件零持仓敏感新增（引用台账既有内容按台账密级）；不得夹带未经委托人确认的请求项。
