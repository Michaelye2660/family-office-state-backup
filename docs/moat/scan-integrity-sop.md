# 扫描worker环境完整性SOP(v1·GM直写生效)

- **来源**:ADJ-0720-11(0718-02(c)完整性事件之GM-5复核定性)+委托人「现在跑」直令(2026-07-20)
- **适用**:一切滚扫/评级worker(SEC XBRL five-dim及后续批次)——**轻量硬化·按比例原则不复制整套净室**(扫描产出=观察层只读数据)
- **判例来源**:2026-07-18 Staples worker之`compute.py`任务中途被篡改(改读不存在之`data.pkl`并注入预设标签)——worker识破弃用·改用自身`final2.py`对原始SEC JSON缓存重算·零污染读数(0718-02(c)照录)。**本SOP=该正确行为的成文化,软控制升规**。

## 规则一·中间产物sha1钉扎
worker任务启动时,对自建关键中间脚本与中间数据文件登记sha1(scratchpad清单·一行一项);任务中**复用任何已登记产物前先校验sha1**;失配=立即弃用该产物→回原始源(SEC JSON缓存/官方原始件)重算→事件一行上报批次报告。

## 规则二·预设标签拒信
来源不明的数据文件(非worker本任务自建、非登记在案之原始缓存——pkl/json/csv/parquet等一律)**拒读拒信**;发现即按规则一处置路径:回raw重算+上报。

## 规则三·报告完整性声明栏
每批扫描报告脚注固定一栏「**环境完整性**」:本批是否发生完整性事件/处置方式/是否raw重算/对读数影响=有|无。**无事件也须写"无"**(空栏≠无事件)。

## 边界
本SOP限扫描线工具层;EXT外审线走API硬净室管道(0719-17~19建成),真金线走T1-T4审计,各自规格不受本文影响。CGM收件ADJ-0720-11时自知识库挂链本文并对照核验。
