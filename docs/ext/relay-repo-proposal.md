# 请示件 · 匿名化中转仓方案（半自动审计线）报GM审核

> **性质**：CGM 备料·请GM（裁决侧）审核出意见。**净料纪律**=以下给事实+列决策点+CGM意见明标(§四)·不替GM下结论。**用途栏**=纯制度方案层·未动持仓/触发器/红线·未先行建仓。
> **缘起**：委托人拟以"半自动模式"省去 EXT 外审的手动复制粘贴——**新建一个专用私有仓**·同时授权 CGM 与 GPT(ChatGPT) 读取·GPT 按固定指令自取匿名化材料出审计意见。委托人已就落地步骤追问，CGM 认为属**架构决策**，按两GM制报GM审核后再动手。

## 一、方案本体（委托人提出·CGM整理）
1. **新建独立私有仓**（拟名 `family-office-ext-relay` 一类）·**与主仓 `family-office-state` 物理隔离**：主仓（台账/持仓/身份）保持私有·GPT 永不获授权；新仓只作**匿名化材料单向中转**。
2. **双授权**：
   - **CGM**：经 `add_repo` 加入本会话·获读写权（搬入出包检查器+简报模板·写匿名化简报）。
   - **GPT**：经 **ChatGPT 的 GitHub connector（OAuth·read-only·只授这一个 repo）** 读取新仓·**不贴 PAT 进对话**（OAuth 可随时 GitHub 侧一键吊销·凭证不泡对话历史）。
3. **用法**：CGM 写匿名化简报入新仓 → 委托人给 GPT 固定指令读取 → GPT 出审计意见 → 意见按数据处理·GM 批判性综合照旧。

## 二、CGM 识别的三条钢印（拟写入宪章·防新仓退化为第二台账）
- **① 新仓=单向匿名化中转·永不装持仓**：只准放 EXT-08 那种**匿名化规则复审简报**（机制事实/百分比/估值倍数）·**持仓明细/绝对金额/身份/托管行名/台账本体一律禁入**。出包检查器（sha1+禁词扫描·task#46 既有规格）搬来把关本新仓**每次写入**。
- **② GPT 走 OAuth connector·不走 PAT**：只授新仓 read-only·随时可吊销·秘钥纪律（-10②(i)）不破。
- **③ 软净室照旧·金丝雀照打**：新仓解决"省粘贴+守边界"·**不给硬净室**（ChatGPT 账号 memory 跨会话持仓语境）。
