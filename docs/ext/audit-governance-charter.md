## 一、异议席升格（EXT线升"常驻异议席"·ADJ-0719-12①）
- **(a) 重大裁决自动送审**：宪法级／方法论修订类 ADJ·管道建成后自动送净室出独立意见·**GM综合逐条回应=硬义务**（采纳与否系GM裁量·终裁在委托人）。
- **(b) 分歧升级权**：异议席意见与GM裁决评级差>一档→**自动升委托人终裁并标红**（既有分歧闸扩适用）。
- **(c) EXT-R 季度纵向审**照旧（见 EXT-R-diet.md）。
- **(d) 审计独立性钢印**："**审计者不入伙**"——异议席**无裁决权、无签名权**·裁决唯一署名制（GM权威四字段）与责任链不变·**三GM制不设**。

## 二、真金审计四档（T1-T4·ADJ-0719-12②③）
| 档 | 触发 | 流程 |
|---|---|---|
| **T1·事前必审** | 新开仓／清仓／单笔≥0.5%净值方案件 | 自动送净室→独立意见→GM逐条回应→**双意见并呈委托人终裁**（互不见稿·分歧>一档标红） |
| **T2·事后周审** | 当周全部已执行动作 | 匿名化汇总送审（"执行与论点一致否·漂移何在"）→意见入周复盘＋喂N14 |
| **T3·预授权豁免** | 已过T1之阶梯设计 | 单批执行不重审（防拖延）·归T2兜底 |
| **T4·委托人点审** | 委托人任何动作/计划前后缀「**审**」 | 无金额门槛/无类型限/**无需理由**·即时送净室→独立意见→GM逐条回应→双呈委托人 |

- **T4两特护**：(i)**计划外动作默认冷却**——凡计划外标记者·T4意见回来前**不执行**（委托人明令「审但先做」可豁免·豁免入档）；(ii)**疑虑登记**——每次"疑的是什么"一行入案·**季度出委托人疑虑命中率读数**（直觉作为仪器被校准·非评判）。

## 三、自动摆渡管道（API制·CGM建设·ADJ-0719-10②·候委托人API key）
- **架构**：CGM侧脚本经 OpenAI API 直连——**净室线=每次全新无状态调用**（结构性净室·无账户/无记忆开关·"可证明干净"由软控制升硬保证·金丝雀降为程序合规确认项如实降级）；**EXT-R线=CGM受控上下文重放**（席位"记忆"=注入清单·EXT-R-diet.md）。
- **四防栏**：(i)**秘钥纪律**=OpenAI API key存CC环境变量·**严禁入库入档**；(ii)**材料边界**（**ADJ-0719-12④相对化修正**）=交易审计包**可含仓位百分比/组合权重/估值倍数**·**仍禁绝对金额/身份信息/托管行名**；(iii)**全程留痕**=每次往返request/response全文归档＋sha1锚定；(iv)**答卷即数据**=回程文本按数据处理·GM批判性综合照旧。
- **候委托人一件事**=**OpenAI API key一枚**（委托人在OpenAI平台生成·配置进CC环境·CGM出配置指引随建设包）；**成本申报**=API按token计费·具体价格候CGM核实报委托人（不编造）。
- **首用**=EXT-08 v2 投放（管道建成后首跑；委托人若不愿等·手动投放照旧可用·两路并行有效）。
- **中转仓线（半自动·规则复审专用·ADJ-0719-13准建）**：独立**中性名**私有仓〔物理隔离主仓·GPT经 **OAuth connector read-only** 授权·CGM经 `add_repo`〕作匿名化材料**单向中转**·**限规则复审软净室用途**（真金 T1-T4／EXT-R 常驻线**不走**·守 API 硬净室；**日落复议钩=API 管道建成满月复盘中转仓存废/降级**·防三线并存固化增生）。**三钢印**=①单向匿名化中转·永不装持仓（出包检查器把关**每次写入**）②GPT 走 OAuth 不走 PAT（可吊销·不破秘钥纪律）③软净室·金丝雀照打（**合格线更新**=除 connector 自然可见信息〔仓名/owner/文件表〕外零其他账户历史泄漏·如实降级不假装标准未变）。**GM增补四件**=(a)**仓名中性化**（驳 `family-office-ext-relay`·改如 `research-review-relay` 类·不含 family-office/资产/审计敏感词·防仓名泄露"family office"语境）；(b)**owner暴露如实申报**（connector 天然使 GPT 见 repo owner 用户名+文件结构·较 paste 法匿名性净降一档·入档为已知暴露面·用户名≠身份细节可接受）；(c)**授权范围探针**（**验收必备·季度复打**·首次连接后固定首问"列出你当前可访问的全部仓库"·答案必**仅中转仓一个**·见主仓/他仓=授权范围事故→立即吊销重配+事件立案·此为最大单点风险〔OAuth 误勾"All repositories"〕唯一现形器）；(d)**装载留痕在主仓**（每次写入何件/sha1/把关结果登记主仓 EXT 档·**中转仓是货舱不是账本**·治理记录不迁移）。**过渡期**（检查器未建）=CGM 手动匿名化把关+逐件申报·**仅限 EXT-08 v2 一件**·第二件起须检查器就位。**README 人读护栏**=新仓 README 明写"本仓仅匿名化规则复审材料·禁一切持仓/身份/绝对金额·违反=事件立案"。

## 四、三层传递内容核验制（ADJ-0719-11①·出包检查器=上线验收必备·无检查器不上线）
- **层1·逐次机械自检**（脚本内置·每次往返强制）：**出包检查器**=发送前 request体 sha1 vs 当期登记 brief（或 EXT-R 注入清单）sha1 **恒等校验**（防出包漂移/中途改写）＋**禁词扫描**（**美元绝对额/账户标识/身份信息类正则**·材料边界防栏机械执行器·**百分比放行·绝对金额拒发**·ADJ-0719-12④）——**任一命中=拒发＋上呈GM**·不静默放行；接收后 response 落盘即 sha1 锚定＋逐字归档。
- **层2·GM周期抽验**：**月度**抽验≥3次往返（或当月总数10%取高）=归档request逐字vs登记版本比读＋禁词复扫＋response完整性核；**EXT-R线另加"注入清单对表"**（实际发送=EXT-R-diet.md登记逐件恒等·防未登记喂料）；**R3季度**全面核（当季全量往返）。核验结果一行入〔M〕·任何不符=管道暂停＋事件立案。
- **层3·委托人查权**：全往返档案在库·sha1锚定·随时可查无需申请——**自动化后可审计性＞人肉摆渡**（复制粘贴无档·管道有档）。
- **双模式全自动确认**：净室线（硬净室）与 EXT-R线（受控上下文）均走本管道全自动·委托人角色自"摆渡人"转"抽查人"·终裁权与查权不变。
