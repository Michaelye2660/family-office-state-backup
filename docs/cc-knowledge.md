## §5 经验教训库（硬教训，每条都付过学费）

1. **"master落后N commits"是幻觉**（两次）：routine会话的origin快照过时所致，master从未真落后——先 git fetch 实测,不要信会话内计数（〔M〕11）。
2. **标记提取要用整行匹配**（两次）："====文档开始===="字样会先出现在指令句子里，正文块在后——按 `strip()==标记` 整行匹配，否则截出3个字符。
3. **N8血案**：曾把专用签名私钥base64打进对话——用户整案作废（v15.34），N8由此而生。秘密值的正确姿势：用户自己生成、产品级秘密管理（环境变量）、清单只记名称与轮换路径（docs/SECRETS-INVENTORY.md）。
4. **API建的trigger缺repo sources与MCP连接器**：生产trigger一律UI建（N7第五款）；发现自己越权建了实验trigger→4分钟内自删+自首（裁决侧定性"越权+模范自首,记过不记罪"）。
5. **版本号冲突照例顺延**：用户/裁决侧给的版本号若已被占用，顺延并在版本行加注，不改不问（已成惯例，v15.42/44/46三例）。
6. **〔M〕条目插入位置要三查**（错过两次）：新条目锚错行会插到旧条目前面——插入后立即读回验证顺序。
7. **会话≠容器**：环境变量轮换后旧容器不热更新，自然回收后自愈；检测协议见SECRETS-INVENTORY.md（只看HTTP状态码，不打印值）。
8. **镜像仓push 403**：备份仓不在会话授权集时用add_repo加入即可，无需clone（push-only）。
9. **上下文压缩无损**：对话很长会自动压缩，制度记忆全在仓库——"对话为草稿，仓库为正本"（N10）不是口号，是本文件存在的理由。
10. **转写稀释不是GM专利,长文过上下文即有损**（CGM-1血案,2026-07-12·〔M〕55）：CGM-1归档EXT第2期时把674行原文压缩成212行,归档注却写"原文一字不改"——定性=转写稀释+不实保真声明,外审归档窗口覆盖查获,CGM-1认领不申辩。GM侧失败模式⑧自此在CGM侧同型实证:两侧共有,非载体特性。正确姿势:聊天粘贴长文归档一律走外审归档窗口（原文直进其上下文,零转写）;"一字不改"声明必须附同会话对源文本计算之sha1,无sha1=声明无效。

## §6 运行手册（继承者第一天用）

- **收件流程**：口令"收件"→ git pull --rebase → 读adj-inbox/ → 三要素校验 → 执行 → 写adj-archive/ADJ-XXXX-receipt.md（编号|项数对应|逐项结果|commit哈希|异常与建议）→ 原件git mv归档 → 台账版本行 → commit+push+镜像 → 向用户报"分支+哈希"。
- **push纪律**：git pull --rebase origin master 先行；被拒绝重试（网络错误指数退避2/4/8/16s）；每次master push后跟一次镜像 `git push https://github.com/Michaelye2660/family-office-state-backup.git master:master`。会话环境若指定工作分支:commit落分支后**默认直接归一master**（〔M〕56常设授权,2026-07-12;仅限快进,永不force-push,内容层授权要件不变）;继任会话完成重建阅读后**自动切master**（`git checkout -B master origin/master`,〔M〕57,告知不请示,cgm-succession§五步4a）。
- **台账纪律**：任何更新必留版本历史行（含类别:机械类/判断类）；头部版本号同步刷新；机械类=单程执行，判断类=全回路（执行+逐项编号回报）。
- **邮件白名单**（settings.json autoMode）：收件人恒等 yxy2660@gmail.com、发件 onboarding@resend.dev、端点仅 api.resend.com/emails、正文只装简报文本、key只从 $RESEND_API_KEY 读。
- **备份三层**（N10）：git历史 / 周镜像备份仓 / 用户本地月度ZIP+季度claude.ai导出。
- **继承者首日清单**：①通读台账〔M〕〔N〕全节；②读本文件+裁决侧知识库；③读最近5份receipt感受回执格式；④读docs/motions/CC-0712-01.md理解两权分立的来历；⑤向用户报到并确认有无仅口头交接的事项。
