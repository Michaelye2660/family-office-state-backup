## 信箱通道v2（Issue投递·2026-07-12设计完成·**暂缓启用——委托人同日裁定"先不弄",现行=复制粘贴模式**;协议与Action schema保留为现成方案,启用只差自定义GPT配置一步）

- **取件**:委托人口令"外审归档"（可不带正文）→CGM经github工具取最新EXT Issue→**原文一字不改**落docs/external-audits/YYYY-MM-DD-EXT-NN.md→Issue下评论归档commit哈希并close（即回执）→〔M〕登记→镜像备份自动覆盖。

## 外审归档窗口（2026-07-12委托人设立·专用claude.ai/code会话"第三方审计报告"）
- **分工**:EXT报告全文粘贴至该专用会话（防长文淹没hub会话历史）,口令"外审归档"→该会话执行:原文一字不改落docs/external-audits/YYYY-MM-DD-EXT-NN.md+正文sha1申报+台账版本行登记+commit+push+镜像。
- **权限边界**:该窗口系委托人驱动会话（非定时,不受治理只读约束）,但职责限定=外审报告归档一件事——**只写docs/external-audits/与台账版本行;红标处置/判断类事项/回应义务一律回hub会话（CGM长期会话）办理**;归档窗口对报告内容不评论、不处置、不代任何一方回应。
- 会话隔离原则:各会话互不可见,仓库为唯一共享层——报告只有落库后hub/GM才可读,"归档"不是可选项而是传递本身。
